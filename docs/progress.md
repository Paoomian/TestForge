## 技术决策

| 决策 | 原因 |
|------|------|
| 子表更新采用删旧插新 | 简化逻辑，避免复杂diff |
| JS脚本使用PyMiniRacer沙箱 | 5秒超时强制中断，安全隔离 |
| 轮询替代WebSocket（1.5s间隔） | 实现简单，WebSocket已实现但未使用 |
| 变量提取Tab移除，统一使用数据规则 | 功能重复，数据规则完全覆盖 |
| 批量执行拖拽排序用HTML5原生API | 零依赖实现 |
| MySQL ENUM改VARCHAR(20) | 避免截断问题 |
| datetime使用naive模式 | 避免timezone偏移错误 |
| 迁移脚本合并为upgrade_all.py | 10个独立迁移合并为1个统一脚本，可重复执行 |
| 场景编排UI用流程图+属性面板 | 分支可视化，编辑空间充足，CSS连线零依赖 |
| 分支归属用branch_of字段推导 | 前端渲染用，不改后端模型，从true_branch/false_branch反推 |
| test_case_extracts合并到test_case_data_rules | 消除冗余表，统一用rule_type="extract"表示提取规则 |
| 编排模式通过率=pass/(pass+fail+error) | skipped是条件跳过的分支，不应计入分母拉低通过率 |
| 编排任务详情用node_tree替代flat列表 | 通过detail.node_id反查SceneNode.suite_id构建树结构 |
| batch-delete路由放在/{run_id}之前 | FastAPI按定义顺序匹配，避免"batch-delete"被当作int解析导致422 |
| Prompt模板合并为单一system_prompt字段 | user_prompt留空由后端自动生成，降低用户配置复杂度 |
| JSON解析预处理fix_json_quotes | 逐字符状态机修复AI返回的未转义引号，比正则更可靠 |
| 外键删除前先解除引用 | 删除Skill前将关联Task的skill_id置NULL，避免约束报错 |

## 项目挑战与技术债务

### 稳定性挑战

| 问题 | 影响 | 优先级 | 解决方案 |
|------|------|--------|----------|
| UI 录制选择器包含动态 class | 用例回放失败 | P0 | 已优化：过滤 focus/active/hover 等动态 class |
| Vue 路由跳转产生多余导航步骤 | 录制步骤冗余 | P0 | 已优化：点击后 2 秒内跳过导航记录 |
| 页面加载时机不确定 | 元素找不到 | P1 | 添加显式等待（wait_for_selector） |
| 截图流延迟 | 用户体验不佳 | P2 | 已优化：频率提升至 100ms |
| 下拉框等交互组件录制困难 | 复杂交互无法录制 | P1 | 需要专门的 select 组件支持 |

### 性能挑战

| 问题 | 影响 | 优先级 | 解决方案 |
|------|------|--------|----------|
| 截图流带宽消耗 | 大页面加载慢 | P2 | 考虑增量压缩或降低分辨率 |
| 步骤截图存储空间 | 数据库膨胀 | P2 | 已优化：不再保存截图到数据库 |
| 大量用例并发执行 | 服务器压力大 | P2 | Celery 并发控制、限流策略 |
| WebSocket 连接管理 | 内存占用 | P3 | 连接池、心跳检测、超时清理 |

### 兼容性挑战

| 问题 | 影响 | 优先级 | 解决方案 |
|------|------|--------|----------|
| Shadow DOM 元素无法定位 | 特定框架用例失败 | P2 | 需要特殊处理 Shadow DOM |
| iframe 内嵌页面录制 | 跨域限制 | P2 | Playwright frame 切换 |
| 移动端 Web 适配 | 响应式布局问题 | P3 | 视口自适应 |
| 不同浏览器差异 | 兼容性问题 | P3 | 多浏览器引擎支持 |

### 代码质量挑战

| 问题 | 影响 | 优先级 | 解决方案 |
|------|------|--------|----------|
| TypeScript 类型定义不完整 | 编译警告 | P3 | 补充类型定义 |
| 错误处理不统一 | 调试困难 | P2 | 统一错误处理机制 |
| 单元测试缺失 | 回归风险 | P2 | 补充核心模块测试 |
| 日志规范不统一 | 排查问题困难 | P3 | 统一日志格式 |

## 项目缺点

### 功能层面

1. **UI 录制局限性**
   - 不支持移动端原生 App 录制
   - 复杂交互（拖拽排序、画布操作）支持有限
   - 录制过程中无法实时编辑步骤

2. **执行能力不足**
   - 不支持分布式执行（多节点并行）
   - 缺少失败自动重试机制
   - 无测试数据管理能力

3. **报告能力有限**
   - 不支持报告导出（PDF/HTML）
   - 无历史趋势对比
   - 缺少报告分享功能

4. **协作能力缺失**
   - 无用例评审流程
   - 不支持多人协作编辑
   - 缺少操作审计日志

### 技术层面

1. **架构限制**
   - 单体架构，扩展性受限
   - 数据库设计未考虑分库分表
   - 缺少缓存策略

2. **部署复杂度**
   - 依赖较多（MySQL + Redis + Celery）
   - 缺少 Docker 容器化支持
   - 无自动化部署流程

3. **监控缺失**
   - 无应用性能监控（APM）
   - 缺少错误告警机制
   - 无资源使用监控

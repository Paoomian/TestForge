# TestForge 后台UI改造优先级清单

## 改造说明

本次改造以「不破坏现有功能」为前提，仅做视觉优化和样式统一。所有改动基于CSS变量系统，通过全局样式覆盖Arco Design组件实现，无需修改业务逻辑代码。

## 不建议改动的部分

以下内容在当前阶段不建议修改，以保证功能稳定性：

| 组件 | 原因 |
|------|------|
| Monaco Editor | 使用独立的vs-dark主题，与整体风格差异较大但功能完整，强行修改可能影响代码编辑体验 |
| 已有页面的功能交互 | 如用例编辑抽屉的4Tab结构、表单验证逻辑等，这些是核心业务流程 |
| API接口调用逻辑 | request.ts中的拦截器、token刷新机制等 |
| 路由配置和权限控制 | router/index.ts中的认证守卫 |
| 状态管理逻辑 | Pinia store中的数据流 |

## 改造收益

1. **视觉统一**: 登录页与后台风格完全一致，淡蓝紫低饱和渐变贯穿始终
2. **组件一致性**: 所有Arco Design组件(按钮、输入框、表格、弹窗等)样式统一
3. **可维护性**: CSS变量系统使得未来调整主题色只需修改`theme.css`一处
4. **零破坏性**: 所有改动仅涉及样式层面，业务逻辑代码未做任何修改
5. **渐进增强**: 可以随时通过修改CSS变量微调整体风格

---

## 如何验证

1. 启动前端: `cd frontend && npm run dev`
2. 访问登录页: 检查样式是否正常
3. 登录后依次访问:
   - 仪表盘: 检查统计卡片、欢迎区域
   - 项目列表: 检查表格样式、操作按钮
   - 接口用例管理: 检查左侧面板、右侧表格
   - 接口调试: 检查请求/响应区域样式
4. 检查各组件交互: 按钮hover、输入框focus、表格行悬浮等
